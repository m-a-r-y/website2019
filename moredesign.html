Mobile payment / checkout

A design challenge to create checkout screens for mobile purchasing. Screens created to add a new payment method, select from saved credit cards, and edit existing payment methods.

Add a new payment method

Note that the user is unable to proceed to review and submit their order until they have entered their credit card information.

User can select how they want to add a new payment method.

Users cannot proceed until they fill out the form.

Users have the option to save the card information for next time.

Select from a saved payment method

Note that the user is still unable to proceed to the next step unless they have selected a payment method.

Users must select a payment method before proceeding.

Users can reselect different methods before proceeding to the next screen.

Edit or delete a payment method

In this edit card feature, the user have the option to edit and/or delete a saved credit card. A CVV entry is require as a secondary step to prevent accidentally deleting a credit card.

Users can choose which cards to edit one at a time.

Users can edit the card info or have the option to delete the card.

Users must enter card CVV before they can delete a card to prevent accidents.

Users can now confirm to delete a card.

Looking ahead to the future!

The next iteration would focus on designing a way to implement a camera feature to automatically scan the front and back of a credit card to autofill the credit card number field.

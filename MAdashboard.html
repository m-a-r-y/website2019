About the Project

Mobile Action feed users information and analytics on an app's performance on the Google Play Store and Apple App Store. All the data provided by Mobile Action analytics can help users become more proactive in making their app gain more visibility and generate more downloads.

The focus of this UI redesign was to simplify the old design and streamlining the process for users to easily optimize their apps.

The design process used in this project was the “Double Diamond.”

DISCOVER • RESEARCH

Data collection

User interviews

User Research

When gathering user research findings, the first step was to understand how the users interact with the current dashboard and its challenges. The goal was to uncover better ways to redesign the dashboard for the users.

Interview questions

REGARDING CUSTOMER ISSUES

What issues do most users have when they call in for support?

How often does that question come up?

How does the issue affect their psyche?

How many issues do users have on average? What is the highest number ever from one user?

How similar are the users issues and questions?

How do these issues affect their overall perception of Mobile Action?

REGARDING PRODUCT HISTORY

What is the highest priority product issue that the team currently have to solve?

How was that issue discovered? How are most issues discovered?

Is pushing new features more important or fixing standing issues? How is it decided?

Problem definition

Defining the Problem

Key Terms Definitions
Users Mobile app developers, marketers, product managers, and others in the app/tech world.
Challenges

1. The navigation bar is not optimized, and the dashboard home display provides an unclear message as to how to best use the product.

2. The current dashboard UI/front-end is outsourced so changes are difficult and slow to change.

3. The color scheme is not appealing.
Solution A product redesign.
Benefits A responsive dashboard that captures all audiences using different devices with varying screen size.
"
